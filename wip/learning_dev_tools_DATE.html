Recently though I have wanted to expand my understanding and writing this blog has given me a great opportunity to learn as I go. I hope software devs can have a laugh at my ignorance and fellow armchair experts can learn something with me.

What gave it away I was lacking critical knowledge? I was only recently taught I should be using a virtual environment per each Python script/program instead of just raw dogging everything globally. Great shame was brought to me, my family, and department when the local Vim wizard learned of this transgression. I am required to atone.

Basic idea going into this: HTML files > VSCode > Git > GitHub > Webhosting

 

Things I learned that you might find useful:

  • Git commands and steps in a modern change control process.

 

Funny things I learned about development:

  • Claude prefers Git CLI, so that's what I learned.
  • Git is really the only version control system anyone seriously uses. Some Redditors tried to say otherwise, but were burried in downvotes.
  • Best practice dictates dev branches should be deleted after merging to main. When branches are merged, all the commits carry over, so they were way more ephemeral that I expected.
  • Not making sure you are on the right branch when making changes means you get to learn new commands to transfer the changes and not lose them :(
  • It feels really silly to create and approve my own pull request after auditing so many change processes that require segregation of duties.
  • Devs are bullied by other devs for using Git CLI - Theater Thug
  • When you clone a repo, it creates a folder for that repo automatically even though you already created a folder with the same name, and it's very confusing when you didn't realize you were in a nested folder and that's why nothing was working.
  • Git and GitHub do not agree on default main/master branch nameing convention.
  • I really don't understand how authentication with VSCode > Git > GitHub works. I just kept running commands until I got an OAuth prompt TBH.
  • GitHub PagesTM is nice for static hosting. Use my referral link if you would like a 75% discount.
