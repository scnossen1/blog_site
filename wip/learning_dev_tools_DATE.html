Things I learned that you might find useful:


Terminology:

  • Repository: A storage location for code, which can be local (VSCode) or remote (GitHub).
  • Commit: A snapshot of your changes in the repository.
  • Branch: A separate line of development in a repository.
  • Merge: Combining changes from different branches.
  • Pull Request: A request to merge changes from one branch to another, typically used in collaborative workflows.

Git Commands:


  • git init - Initialize a new Git repository: Creates a new LOCAL .git directory in your project. This is what actually defines if something is a Git repository or not.
  • git clone [url] - Clone an existing repository: Copies a remote repository to your local machine. Useful if you want to work on an existing project.
  • git add [file] - Stage changes for commit: Lets you decide what files to include in the "commit" stage
  • git commit -m "message" - Commit staged changes with a message: Records changes in your LOCAL repository
  • git push - Push changes to a REMOTE repository.
  • git pull - Pull changes from a REMOTE repository: Useful if you need get a current version from GitHub or another source.
  • git branch - List, create, or delete branches: Manages branches in your repository. For me, it's useful for holding work before it "goes live".
  • git checkout - Switch to a different branch: Switch between LOCAL branches you have created or pulled down.
  • git merge - Merge two LOCAL branches: E.g. 'git merge feature-branch' while the main branch is checked out will merge feature-branch into main.
I don't touch on a lot of the commands for troubleshooting, but the ones I found usefull were 'git status' and 'git log'.
